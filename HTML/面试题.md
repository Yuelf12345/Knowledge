## meta 标签
<meta> 标签是HTML文档中用于提供元数据的标签，这些数据不会显示在页面上，但会被浏览器、搜索引擎或其他网络服务解析和使用。
```html
    <head>
        <!-- 编码 -->
        <meta charset="UTF-8">
         <!-- 移动设备优化  -->
        <meta name="viewport" content="width=device-width, initial-scale=1.0, maximum-scale=1.0, user-scalable=no">
        <!-- 禁止缩放 -->
        <meta name="format-detection" content="telephone=no">
        <!-- 苹果设备优化 -->
        <meta name="apple-mobile-web-app-capable" content="yes">
        <!-- 苹果设备主题色 -->
        <meta name="theme-color" content="#4285f4">
        <!-- SEO相关 -->
        <meta name="description" content="页面描述">
        <meta name="keywords" content="关键词">
        <!-- 社交媒体优化 -->
        <meta property="og:title" content="标题">
        <meta property="og:description" content="描述">
        <meta property="og:image" content="图片URL">
        <title>页面标题</title>
    </head>
```

## link标签和style标签
HTML中的 <link> 和 <script> 标签都是用于引入外部资源的元素，但它们有不同的用途和特性。让我详细解释一下它们的区别：
<link rel="stylesheet" href="styles.css">
    rel: 定义当前文档与被链接文档的关系（如stylesheet、icon等）
        Preload：预加载资源，加载当前页面必须的资源
        Prefetch：预获取资源，加载当前页面可能会用到的资源（浏览器空闲时加载）
    href: 外部资源的URL
    type: 资源的MIME类型（如text/css）
    media: 指定媒体类型（如screen、print）
特点
    异步加载: CSS文件默认异步加载，不会阻塞HTML解析
    无脚本: 不能包含JavaScript代码
    放在head中: 通常放在<head>标签内
<script> 标签
常见属性
src: 外部脚本文件的URL
type: 脚本的MIME类型（默认为text/javascript）
async: 异步加载脚本（加载完立即执行）
defer: 延迟执行脚本（在DOM解析完成后执行）
特点
阻塞渲染: 默认情况下，脚本会阻塞HTML解析和渲染
可内联: 可以直接在标签内编写JavaScript代码
可控制加载行为: 通过async和defer属性控制加载和执行时机
